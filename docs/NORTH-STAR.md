# NORTH-STAR.md — 锚点文件

> 任何文档或功能变更前，对照此文件确认三点未变。

---

## 终极目标（三轴）

### 轴一：AI 全域工作能力
AI Agent 能在我的任意项目目录中运行，拥有完整的文件系统读写权限和项目上下文。
当前实现：Claude Code via `cc:nexus` 容器，WORKSPACE_ROOT 整体挂载。

### 轴二：零摩擦上下文同步
无论我在哪里、用什么设备，都能流畅地将真实世界的上下文（文字、图像、文件、语音）同步给 AI，
并以最低的操作成本获取 AI 的响应——包括但不限于浏览器终端、移动端 App、IM 频道（Telegram）。

### 轴三：极致的 Agent 管理体验
所有正在运行的 AI Agent 和会话都有统一、灵活的管理界面：查看状态、切换上下文、
启动/停止任务，在 PC 端和移动端都达到极致的交互体验。

---

## 目标用户

**用户即开发者本人**（单用户/个人服务器）
- 同时运行多个 Claude Code Agent，需要随时从任意设备查看进度和干预
- 不想保持 SSH 连接，浏览器/App 关掉后 Agent 继续运行
- 希望在走路、坐地铁时通过 Telegram 给 AI 下任务，回家后在 PC 上接续

---

## 明确不做的事（Out of Scope）

- **不替换 tmux**：Session 持久化、scrollback 全部由 tmux 负责
- **不做多用户系统**：单密码 JWT，不做注册/权限/团队功能
- **不做通用 Web SSH**：功能边界是 claude CLI 工作流，非通用终端工具
- **不暴露 Docker socket 到前端**

---

## 变更保护规则

修改任何功能或文档前，确认以下三点未被违反：
1. 该变更是否增强了「AI 全域工作能力」「零摩擦上下文同步」「极致 Agent 管理」三轴之一？
2. 是否仍然是单用户场景设计，没有引入团队/多用户复杂度？
3. Out of Scope 的四条是否依然成立？

如有冲突，暂停并告知用户。
